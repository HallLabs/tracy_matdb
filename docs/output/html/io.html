File Conversion and I/O Routines¶

We use quippy and ase for all our calculation and data needs. However, calculations are initially performed in VASP for all configurations. This necessitates conversion between file types, etc.

I/O Functions¶

Functions for interacting with various output formats.

matdb.io.atoms_to_cfg(atm, target, config_id=None, type_map=None)[source]¶

Converts an matdb.atoms.Atoms object to a cfg file.

Parameters:
  • atm (matdb.atoms.Atoms) – the atoms object.
  • target (str) – path to the cfg file to be written.
  • config_id (str) – the config id for the atoms object.
  • type_map (dict) – a type map to match the species to a larger system than present in the system.
matdb.io.cfg_to_atomslist(cfgfile, config_type=None, species=None)[source]¶

Converts the CFG format file to an internal AtomsList object.

Parameters:
  • cfgfile (str) – path to the file to convert.
  • config_type (str) – name of the config_type to assign to each configuration.
  • species (list) – list of element names corresponding to the integer species in the CFG dictionary.
Returns:

An AtomsList object containing the all the cells in the CFG file.
Return type:

matdb.atoms.AtomsList
matdb.io.cfg_to_xyz(cfgfile, outfile='output.xyz', config_type=None, species=None)[source]¶

Converts MTP’s CFG forrmat to XYZ.

Note

Multiple frames in the CFG file will be converted to multiple frames in the XYZ file.

Parameters:
  • cfgfile (str) – path to the file to convert.
  • config_type (str) – name of the config_type to assign to each configuration.
  • species (list) – list of element names corresponding to the integer species in the CFG dictionary.
matdb.io.is_link(obj)[source]¶

Determines whether the specified object is a link according to the matdb templating specification.

matdb.io.load_dict_from_h5(h5file, path='/')[source]¶

Reads an open hdf5 file into a dictionary.

Parameters:
  • h5file (file) – the h5 file to be read.
  • path (str, optional) – the path within the h5 file presently being read. Default is ‘/’.
Returns:

a dictionary containing the contents of the h5 file.
Return type:

dict
matdb.io.order_stress(xx=None, yy=None, zz=None, yz=None, xz=None, xy=None)[source]¶

Ensures that the six unique components of the stress tensor are presented in the correct order assumed by ASE.

matdb.io.read(context, yfile)[source]¶

Reads in the specified YAML file, following any additional file directives to compile a full representation of the template hierarchy for the root file.

Parameters:
  • context (str) – path to the root folder where the yaml file is located. Needed for relative paths of file links.
  • yfile (str) – name of the template YAML file relative to context. Should not include the .yaml or .yml extension.
matdb.io.save_dict_to_h5(h5file, dic, path='/')[source]¶

Saves a nested dictionary to an open hdf5 file.

Parameters:
  • h5file (file object) – the h5 file to be saved to.
  • dic (dict) – the dictionary to save.
  • path (str, optional) – the path within the h5 file that the dict will be saved to. Default is ‘/’.
matdb.io.symmetrize(xx=None, yy=None, zz=None, yz=None, xz=None, xy=None)[source]¶

Returns the 3x3 stress matrix from the specified components.

Note

the components are: xx yy zz xy yz zx.

matdb.io.vasp_to_xyz(folder, outfile='output.xyz', recalc=0, properties=['species', 'pos', 'z', 'dft_force'], parameters=['dft_energy', 'dft_virial'], config_type=None)[source]¶

Creates an extended XYZ file for the calculated structure in OUTCAR for the given folder.

Parameters:
  • folder (str) – path to the folder to convert.
  • outfile (str) – name of the XYZ file to create. The file will be created in the same folder as the original if no absolute path is given.
  • recalc (bool) – when True, re-convert the OUTCAR file, even if the target XYZ file already exists.

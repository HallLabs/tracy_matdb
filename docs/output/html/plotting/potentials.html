Potentials Functions¶

Functions for plotting interatomic potential performance vs. correct (REF) answers within a particular matdb.atoms.AtomsList.

matdb.plotting.potentials.EvsV(pot, atoms, folder=None, base64=False, index=None, valkey='ref')[source]¶

Produces an energy vs. volume plot for the energies predicted by the potential.

Parameters:
  • pot (matdb.calculators.basic.AsyncCalculator) – IP to calculate properties with.
  • atoms (matdb.atoms.AtomsList) – list of atoms from which to calculate correlations.
  • folder (str) – path to the folder where the saved image should be stored.
  • base64 (bool) – when True, use a base-64 encoded src for the output of this image; otherwise, the image is saved to folder and only its name is returned.
  • index (int) – integer index of this plot in the parent collection.

matdb.plotting.potentials.generate(plots, pot, atoms, folder=None, base64=False, index=0, ndimer=50, ntrimer=75, valkey='ref')[source]¶

Generates a set of plots for a given potential and atoms list. The following are possible options; for each one, a shorthand character is specified:

  1. Energy Correlation Plot (e)
  2. Force Correlation Plot (f)
  3. Virial Correlation Plot (v)
  4. Energy vs. Volume Plot (o)
  5. Dimer Plots for 2-body Potentials (d)
  6. Trimer Plots for 3-body Potentials (t)

You can specify which plots to include in the set by adding the character codes to the plots argument. For example, to plot dimer, trimer and energy vs. volume plots, use plots=”dte”.

Note

The specified order of the plots will be the same order they will show up in the interactive plot.

Parameters:
  • plots (str) – string of character codes as described above.
  • fit (matdb.calculators.basic.AsyncCalculator) – trainer that has an IP to calculate properties with.
  • atoms (matdb.atoms.AtomsList) – list of atoms from which to calculate correlations.
  • folder (str) – path to the folder where the saved image should be stored.
  • base64 (bool) – when True, use a base-64 encoded src for the output of this image; otherwise, the image is saved to folder and only its name is returned.
  • seed_index (int) – integer index to use for the first plot. It is incremented automatically for each plot added.
  • ndimer (int) – number of samples to take along the dimer trajectory.
  • ntrimer (int) – number of samples to take along the trimer angle trajectory.
  • valkey (str) – prefix on parameter and property names that should be used for comparison.

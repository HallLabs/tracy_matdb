Training Controller Documentation¶

Since some potential fitting requires multi-step processes, we need to implement Controller objects that keep the various training steps ordered correctly, and provide easy access to relevant data from previous steps. This module supplies objects (similar in spirit to matdb.database.__init__) for producting sequences of training objects that can be repeated across parameter grids.

Controller for handling sequences of training steps.

Parameters:
  • db (matdb.database.Controller) – controller for the available data sets to use.
  • root (str) – path to the root directory for the system.
  • fits (dict) – key-value pairs defining the list of fitters to try out.
  • e0 (list) – list of float values indicating the isolated atom energy in eV.
  • plotting (dict) – keyword arguments for plotting to generate for each trainer after it is finished fitting. This uses generate().
  • kwargs (dict) – additional key-value pairs to pass down to the individual trainer objects.

find(pattern)[source]¶

Looks for data in the database. Either with the specific identifier or with a regular expression (to find multiple based off a pattern).

Finds a list of trainers that match the specified pattern. Trainer FQNs follow the pattern trainer-suffix.step where the suffix may be optional and the step names are defined by the name of each Trainer sub-class. Wildcard * or any other fnmatch() patterns may be used.

class matdb.fitting.controller.TrainingSequence(name, repeater, root, controller, steps, **kwargs)[source]¶

Represents a sequence of training steps (each sub-classing Trainer) where each subsequent step depends on the results of the previous step.

Parameters:
  • name (str) – name of the training sequence so it can be referenced by FQN notation.
  • repeater (TSequenceRepeater) – repeater for a set of training sequences that all share the same root training and validation databases.
  • root (str) – root directory in which all other training sequences for the configurations in the same specification will be stored.
  • controller (TController) – instance controlling multiple trainers.
  • steps (list) – list of dict describing the kinds of potential training steps to setup.
  • kwargs (dict) – key-value pairs to pass to the individual trainer objects (only used if they don’t override the values).
